## ⚠️ KNOWN ISSUES

### Docker Daemon Not Running (Windows)
**Status**: Infrastructure issue, not code issue
**Workaround**: 
1. Open Docker Desktop GUI manually
2. Wait for daemon to initialize (check tray icon)
3. Then run: `docker-compose up -d`
4. Or use: `scripts\deploy.bat`

**Why This Happens**: WSL2 backend initialization timing on first launch

**Resolution**: Single manual start typically fixes; Docker will auto-start on future sessions

## 🚀 QUICK START GUIDE

### Prerequisites
- Docker Desktop (Windows/macOS) or Docker + docker-compose (Linux)
- Python 3.11+
- CMake 3.20+ (for C++ compilation)
- Visual Studio 2022/2019 or MinGW (Windows) / GCC (Linux) / Clang (macOS)

### Step 1: Install Dependencies
```bash
# Choose one:
pip install -r requirements-base.txt -r requirements-cpu.txt      # CPU
pip install -r requirements-base.txt -r requirements-gpu.txt      # GPU (requires RAPIDS conda)

# Or use conda for GPU:
conda create -n medallion -c rapidsai -c conda-forge -c nvidia rapids=24.04 python=3.11
conda activate medallion
```

### Step 2: Start Infrastructure
```bash
# Automated (recommended):
./scripts/deploy.sh                    # Linux/macOS
.\scripts\deploy.bat                   # Windows

# Manual:
docker-compose up -d
python scripts/check_infrastructure.py
```

### Step 3: Build C++ Engine
```bash
./scripts/build_execution_engine.sh    # Linux/macOS
.\scripts\build_execution_engine.bat   # Windows
```

### Step 4: Run Tests
```bash
pytest tests/test_infrastructure_integration.py -v
```

### Step 5: Benchmark GPU
```bash
python scripts/benchmark_gpu.py --size 10 --iterations 3
```

## 📞 SUPPORT

### Docker Issues
- Check context: `docker context ls` (should be `desktop-linux`)
- Restart Docker Desktop
- Check port availability: `netstat -ano | findstr :9000` (Windows)

### Build Issues
- Verify CMake: `cmake --version`
- Check compiler: `cl.exe` (MSVC) or `g++ --version` (GCC)
- CUDA detection: `nvcc --version`

### GPU Issues
- Verify NVIDIA driver: `nvidia-smi`
- Check PyTorch CUDA: `python -c "import torch; print(torch.cuda.is_available())"`
- Rebuild if needed: `pip install torch --index-url https://download.pytorch.org/whl/cu121 --force-reinstall`

### Python Issues
- Ensure in project root: `cd e:\PRO\JIM_Latest`
- Add to PYTHONPATH: `set PYTHONPATH=%PYTHONPATH%;e:\PRO\JIM_Latest`
- Check version: `python --version` (should be 3.11+)
